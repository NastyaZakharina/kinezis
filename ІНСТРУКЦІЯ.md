## 🌐 Сайт

**Адреса:** https://nastyazakharina.github.io/kinezis

Сайт — це звичайні HTML-файли, які зберігаються на **GitHub** (безкоштовно) і автоматично публікуються через **GitHub Pages**.

### Як оновити сайт
Всі зміни робляться через Claude Code на твоєму комп'ютері → автоматично потрапляють на GitHub → через ~1 хвилину сайт оновлюється.

### Основні файли
| Файл | Що це |
|------|-------|
| `index.html` | Головна сторінка |
| `catalog.html` | Каталог товарів |
| `product.html` | Сторінка товару |
| `about.html` | Про нас |
| `contacts.html` | Контакти |
| `data.js` | База даних товарів (ціни, фото, описи) |
| `style.css` | Зовнішній вигляд сайту |

### Як додати або змінити товар
Відкрий `data.js` — там список всіх товарів. Кожен товар виглядає так:
```
{ id: 'назва', name: 'Назва товару', price: 12000, image: 'посилання на фото', ... }
```

## 🤖 Telegram-бот (@Kineziss_bot)

Бот написаний на **Python** і виконує такі функції:

### Що вміє бот
- **Приймає замовлення** — коли клієнт натискає "Замовити" на сайті, бот запитує ім'я, телефон і коментар
- **Сповіщає менеджерів** — одразу після замовлення всі менеджери отримують повідомлення з даними клієнта
- **Зберігає замовлення** — всі замовлення записуються у файл `orders.csv` (для ремаркетингу)
- **Відповідає на FAQ** — автоматично відповідає на часті питання
- **Пересилає невідомі питання** — якщо клієнт пише щось незрозуміле, менеджер отримує це питання і може відповісти через Reply
- **Привітання** — вдень каже "Ми онлайн!", вночі — пише робочий графік

### Як відповідати клієнту через бота
1. Клієнт пише питання в бот
2. Ти отримуєш повідомлення з підписом "❓ ПИТАННЯ від клієнта"
3. **Натискаєш Reply** (відповісти) на це повідомлення → пишеш відповідь
4. Клієнт отримує: "💬 [Твоє ім'я] з Кінезіс відповідає: ..."

### Як стати менеджером
Напиши в бот: `/addmanager kinezis2024`

### Робочі години (бот знає сам)
- Пн–Пт: 9:00–18:00
- Сб: 10:00–15:00
- Нд: вихідний

## 🚂 Railway (хостинг для бота)

**Посилання:** https://railway.app

### Навіщо Railway?
Telegram-бот має працювати **постійно** — навіть коли твій комп'ютер вимкнений. Railway — це хмарний сервер, який тримає бота живим 24/7.

### Як це працює
1. Код бота зберігається на GitHub (файл `telegram_bot/bot.py`)
2. Railway підключений до GitHub і **автоматично** оновлює бота коли ти зберігаєш зміни
3. Бот працює на серверах Railway постійно

### Статуси в Railway
- 🟢 **Online / Active** — бот працює, все добре
- 🔴 **Crashed** — бот впав, потрібно перевірити логи
- 🔵 **Building** — Railway оновлює бота (займає ~2 хвилини)

### Де дивитись логи (якщо щось не так)
Railway → твій проєкт → worker → Deployments → View logs

### Змінні середовища (Variables)
Це секретні налаштування які не зберігаються у відкритому коді:
- `BOT_TOKEN` — токен бота від @BotFather
- `MANAGER_IDS` — твій Telegram ID (отримала коли писала `/addmanager`)

## ❓ Часті питання

**Чому в GitHub є "Danger zone"?**
Це стандартна секція GitHub для будь-якого підключеного застосунку. Там є кнопки "Suspend" (призупинити) і "Uninstall" (видалити Railway App). Нічого натискати не потрібно — це просто попередження що такі дії незворотні. Небезпеки немає.

**Скільки коштує Railway?**
Безкоштовний тріал — 30 днів або $5. Після цього план Hobby — $5/місяць (необмежено). Бот вкладається в безкоштовний ліміт при помірному навантаженні, але надійніше взяти Hobby.

**Чи втрачуються замовлення якщо бот перезапускається?**
Замовлення зберігаються у файл `orders.csv`. Але Railway при кожному деплої може скидати файли. Для надійності варто підключити Google Sheets або Railway Volume (окреме завдання).

**Як поміняти пароль менеджера?**
В Railway → Variables → змінити `MGR_PASS` на новий пароль.
